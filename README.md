# Budget Management System

A comprehensive full-stack budget management application for school budget tracking with real-time monitoring, forecasting, and intelligent alerts.

## Features

- **User Authentication**: JWT-based authentication with role-based access control
- **Budget Management**: Create and manage fiscal year budgets with line items
- **Real-time Monitoring**: Track budget burn rate and spending patterns
- **Variance Analysis**: Compare budgeted vs actual spend with detailed insights
- **Forecasting**: AI-powered predictions of budget overruns and trends
- **Supplemental Budgets**: Support for budget realignment and supplemental allocations
- **Cost Per Student**: Calculate and track cost efficiency per student
- **Asset Management**: Track asset lifecycle and end-of-life projections
- **Intelligent Alerts**: Real-time notifications for budget thresholds and anomalies
- **Commitment Tracking**: Monitor purchase orders and vendor commitments
- **Smart Recommendations**: AI-generated cost-saving and cash-flow optimization suggestions
- **Comprehensive Dashboard**: 5+ metrics visualization with interactive charts

## Tech Stack

- **Frontend**: React 18 + Vite + Recharts
- **Backend**: Node.js + Express
- **Database**: MongoDB Atlas
- **Authentication**: JWT + bcrypt
- **API Client**: Axios

## Quick Start

### Prerequisites

- Node.js 16+
- MongoDB Atlas account
- npm or yarn

### Installation

1. Clone the repository
2. Install all dependencies:
```bash
npm run install:all
```

3. Configure environment variables:

**Backend `.env`** (innuBudgetApp/backend/.env):
```
MONGO_URI=mongodb+srv://<username>:<password>@cluster0.example.mongodb.net/budgetApp?retryWrites=true&w=majority
JWT_SECRET=your-secret-key-change-in-production
PORT=5000
NODE_ENV=development
```

**Frontend `.env.local`** (innuBudgetApp/frontend/.env.local):
```
VITE_API_URL=http://localhost:5000/api
```

### Running the Application

#### Development Mode (Both Frontend & Backend)
```bash
npm run dev
```

#### Backend Only
```bash
npm run backend
```

#### Frontend Only
```bash
npm run frontend
```

#### Seed Comprehensive Demo Data
```bash
npm run seed
```

This resets and repopulates demo data across users, budgets, line items, transactions, alerts, forecasts, commitments, supplemental budgets, recommendations, assets, and contracts.

Demo credentials generated by seeder:
- `admin1@budget.local` / `AdminPass123!`
- `admin2@budget.local` / `AdminPass123!`
- Department head format: `<department>.<school>@budget.local` / `DeptHead123!`
	- Example: `it.sis@budget.local` / `DeptHead123!`

The application will be available at:
- **Frontend**: http://localhost:3000
- **Backend**: http://localhost:5000

### Build for Production
```bash
npm run build:all
```

## API Endpoints

### Authentication
- `POST /api/auth/register` - Register new user
- `POST /api/auth/login` - Login user
- `GET /api/auth/profile` - Get user profile

### Budget Management
- `POST /api/budget/create` - Create new budget
- `GET /api/budget/list` - Get all budgets
- `GET /api/budget/:id` - Get budget details
- `POST /api/budget/line-item/add` - Add line item
- `GET /api/budget/line-items/list` - Get line items
- `POST /api/budget/transaction/record` - Record transaction
- `PUT /api/budget/:budgetId/approve` - Approve budget
- `PUT /api/budget/:budgetId/lock` - Lock budget
- `GET /api/budget/:id/dashboard` - Get dashboard metrics
- `GET /api/budget/alerts/list` - Get active alerts

### Analytics
- `GET /api/analytics/:budgetId/variance` - Variance analysis
- `GET /api/analytics/:budgetId/quarterly-comparison` - Quarterly trends

### Forecasting
- `POST /api/forecast/generate` - Generate forecast
- `GET /api/forecast/list` - Get forecasts

### Supplemental Budget
- `POST /api/supplemental/create` - Create supplemental budget
- `PUT /api/supplemental/:id/approve` - Approve supplemental budget
- `GET /api/supplemental/list` - Get supplemental budgets

### Assets
- `POST /api/assets/create` - Create asset record
- `GET /api/assets/list` - Get assets
- `GET /api/assets/end-of-life` - Get end-of-life insights
- `PUT /api/assets/:assetId/status` - Update asset status

### Recommendations
- `POST /api/recommendations/:budgetId/generate` - Generate recommendations
- `GET /api/recommendations/list` - Get recommendations
- `PUT /api/recommendations/:id/status` - Update recommendation status

### Commitments
- `POST /api/commitments/create` - Create commitment (PO)
- `GET /api/commitments/list` - Get commitments
- `PUT /api/commitments/:id/status` - Update commitment status

## Key Classes & Database Models

- **User**: User accounts with roles (admin, department_head) and department assignment
- **Budget**: Fiscal year budgets with enrollment data
- **LineItem**: Budget line items with allocated amounts
- **Transaction**: Actual spending records
- **SupplementalBudget**: Additional budget allocations
- **Forecast**: Predicted spending and overrun estimates
- **Asset**: IT equipment tracked with lifecycle
- **Alert**: System alerts for budget thresholds
- **Commitment**: Purchase orders and vendor contracts
- **Recommendation**: AI-generated suggestions
- **Contract**: Long-term vendor contracts

## Usage Example

### 1. Register & Login
- User registers with email, password, school, and role
- JWT token issued upon successful login

### 2. Create Budget
- Select fiscal year and enter student enrollment
- System calculates cost per student

### 3. Add Line Items
- Add budget categories (Hardware, Software, etc.)
- Allocate amounts for each line item

### 4. Record Transactions
- Log spending as it occurs
- System automatically calculates burn rate
- Triggers alerts if thresholds exceeded

### 5. View Dashboard
- Monitor budget status and spending trends
- Analyze variance between budgeted and actual
- Review forecasts and recommendations

### 6. Manage Supplemental Budgets
- Create supplemental allocations when needed
- Original budget preserved for audit purposes

## Alert Types

- **Threshold Exceeded**: Budget utilization >80%
- **Unusual Spike**: Anomalous spending detected
- **Underutilized**: Budget <50% utilized in Q3
- **Forecast Overrun**: Projected overrun by end of year

## Forecasting Algorithm

The system uses historical spending data to predict:
- Current monthly spend rate
- End-of-year projected spending
- Variance from budget
- Spending trends (increasing/decreasing/stable)
- Confidence level based on transaction history

## Error Handling

- Input validation on all forms
- JWT token expiration handling
- MongoDB connection error recovery
- API error responses with meaningful messages
- Frontend error boundaries for graceful degradation

## Production Deployment

1. Build frontend and backend
2. Configure MongoDB Atlas credentials
3. Set JWT_SECRET to secure value
4. Deploy backend to Node.js hosting (e.g., Heroku, AWS)
5. Deploy frontend to static hosting (e.g., Vercel, Netlify)
6. Update VITE_API_URL to production backend URL
